# Airflow, Kinesis, Firehose, and S3 Output

## Purpose

This section adds the runtime data movement portion of the project.

The pipeline uses Amazon MWAA / Apache Airflow to run a DAG that retrieves API data and writes it into Amazon Kinesis Data Streams. Amazon Data Firehose then reads from the Kinesis stream and delivers the records into Amazon S3.

```text
Airflow DAG
      ↓
API user posts data
      ↓
Kinesis Data Stream
      ↓
Firehose Delivery Stream
      ↓
S3 user-posts output
```

## AWS Resources Created

| Resource | Name | Purpose |
|---|---|---|
| S3 bucket | `jennyarias-airflow-user-posts-data-v1` | Stores delivered user post records |
| Kinesis Data Stream | `user-posts-data-stream` | Receives records written by the Airflow DAG |
| Firehose Delivery Stream | `user-posts-delivery-stream` | Delivers stream records into S3 |
| MWAA Environment | `jenny-airflow-kinesis-v1` | Runs the Airflow DAG |
| MWAA IAM Policy | `MWAAWriteUserPostsKinesisStream` | Allows Airflow to write records to Kinesis |
| Firehose IAM Role | `firehose-user-posts-s3-role` | Allows Firehose to read from Kinesis and write to S3 |

## Airflow DAG

The DAG used for this section is:

```text
load_api_aws_kinesis
```

The successful DAG run included three tasks:

```text
get_api_userId_params
extract_userposts
write_userposts_to_stream
```

## What Each Task Does

### get_api_userId_params

This task determines the API user ID parameter used for the API request.

### extract_userposts

This task calls the API and extracts user post data.

### write_userposts_to_stream

This task writes the extracted records into the Kinesis Data Stream.

## Kinesis and Firehose Flow

Kinesis Data Streams receives records from the Airflow DAG.

Firehose reads from the Kinesis stream, buffers records, and writes them to S3.

Firehose does not always write each record immediately. It can buffer records by time or size before delivering a file to S3.

## S3 Output

Firehose delivered output to:

```text
s3://jennyarias-airflow-user-posts-data-v1/user-posts/
```

The generated file name looked similar to:

```text
user-posts-delivery-stream-1-2026-07-05-...
```

This file name is generated by Firehose. It is normal for Firehose output files to have system-generated names instead of friendly names like `posts.json`.

The folder structure includes date and hour values, for example:

```text
user-posts/2026/07/05/18/
```

This partition-style path is useful because later tools such as Athena can query or organize data by time.
